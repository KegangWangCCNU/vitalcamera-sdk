Vital Camera — emotion calibration

Capture your resting-face baseline to remove the model's per-class bias.

How it works: stay still with a neutral expression for 5 seconds. We record the model's softmax distribution at rest and call that the baseline. At runtime two distributions are mixed linearly: output = 0.3 · pBase + 0.7 · pCal. The calibrated component pCal peaks at 100 % Neutral when the current distribution exactly matches the baseline (KL divergence = 0); as the KL grows it hands probability mass to whichever classes actually rose above their baseline. The 0.3 / 0.7 weights and τ = 0.6 are fixed and baked into the SDK — pass the resulting baselineLogits array to new BrowserAdapter({ emotionCalibration: { images: [...base64...] } }) (use this page only for inspecting the bias and τ math — the SDK itself only accepts a list of face images, not pre-computed logits).
